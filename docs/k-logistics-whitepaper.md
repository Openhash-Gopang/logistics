## 1. 개요

K-Logistics는 고팡(Gopang) 생태계의 물류 전담 AI 에이전트입니다.

기존 물류 플랫폼이 "새로운 차량을 배차"하는 방식과 달리, K-Logistics는 **이미 이동 중인 차량의 동선에 화물을 끼워 넣는** 방식으로 동작합니다. 이를 통해:

- 빈 차로 이동하는 낭비를 제거
- 추가 이동 없이 자연스러운 수요 충족
- 물류비 평균 **34% 절감**

K-Logistics는 1·2·3차 산업 전 영역과 공공 물류를 단일 AI 플랫폼으로 통합합니다.

## 3. K-Logistics 핵심 메커니즘

### 3.1 동선 겹침 매칭 (Route Overlap Matching)

고팡 블랙박스를 탑재한 모든 등록 차량은 **1분 단위**로 현재 위치·목적지·예상 경로를 Openhash L1 서버에 전송합니다.

새로운 배송 수요가 발생하면 AI는:

```
STEP 1. 수요 분석
  - 품목·수량·중량·온도 조건 파악
  - 출발지·목적지·시간 조건 파악
  - 동선 범위 판별 (L1~L4)

STEP 2. 동선 겹침 차량 탐색
  - PDV L1 이상 인증 차량만 대상
  - 화물 유형 적합 차량 필터링
  - 최근접 L1 서버부터 탐색

STEP 3. 릴레이 조합 (장거리 시)
  - 중간 거점(물류허브, 고속도로 휴게소)에서
    차량 인수인계 2대+ 조합

STEP 4. 결과 제시 + PDV 기록
```

### 3.2 릴레이 인수인계

장거리 배송 시 단일 차량 대신 여러 차량이 구간을 나누어 화물을 이어받습니다.

```
예시: 부산 해운대 → 서울 강남 (냉동 수산물 50kg)

차량A: 부산 해운대 → 천안삼거리휴게소 (260km)
      OpenHash 인수인계 기록
차량B: 천안삼거리휴게소 → 서울 강남 (120km)
      OpenHash 배달 완료 기록
```

## 4. Openhash 계층 구조

```
L1 — 읍면동 서버   : 가장 가까운 차량 탐색, 로컬 수요 처리
L2 — 시군구 서버   : L1 미충족 시 확장
L3 — 광역시도 서버 : L2 미충족 시 확장
L4 — 국가 서버     : 광역 간 이동
L5 — 글로벌 서버   : 해외 수출입
```

**탐색 우선순위 원칙:**
- 수요 동선이 L1 이내 → L1에서만 탐색 (서버 과부하 방지)
- 동선이 L2 이내 → L1→L2 탐색
- 광역 간 → L4까지 탐색

## 5. PDV 기반 배송 증빙

### 5.1 PDV (Private Data Vault)

모든 고팡 사용자는 자신만 접근 가능한 개인 데이터 금고를 보유합니다. K-Logistics의 모든 배송 이벤트는 **6하원칙(5W1H)**에 따라 PDV에 기록됩니다.

### 5.2 OpenHash 불변 기록

배송의 모든 단계는 OpenHash 네트워크에 불변 기록됩니다:

| 이벤트 | 기록 내용 |
|--------|----------|
| 배송 요청 | 품목·수량·중량·출발지·목적지·GDC 예상 |
| 수령 출발 | 차량·운전자 신원·수령 확인 |
| 릴레이 인수인계 | 중간 거점·인수 차량·화물 상태 |
| 냉장 온도 이탈 | 요구 온도·실제 온도·위치·시각 |
| 배달 완료 | 수취인 확인·GDC 결제 |
| 상호 평가 | 점수·코멘트 (위변조 불가) |

## 6. GDC 자동 결제

- **1 GDC ≈ 1,000원** (고팡 디지털 화폐)
- 배달 완료 즉시 자동 이체 (현금 거래 없음)
- K-Tax 연동으로 세금계산서 자동 발행

### 요금 체계

| 구분 | 기본요금 | 거리 추가 |
|------|---------|----------|
| 소화물 (5kg 이하) | 2 GDC | 0.3 GDC/km |
| 일반 화물 | 2 GDC | 0.3 GDC/km |
| 냉장/냉동 | 3 GDC | 0.5 GDC/km |
| 위험물 | 5 GDC | 0.8 GDC/km |
| 중량 추가 | — | 100kg 초과 시 0.1 GDC/10kg |

## 8. 산업별 적용

### 8.1 1차 산업 — 농수산축산물

- 수확 예측 AI가 출하 2주 전 냉동·냉장 창고 선점
- 도매시장·소매점 배분 자동 조율
- **폐기율 최대 62% 절감** (신선도 유지)
- 제주 감귤·당근·옥돔 등 도서 지역 특산물 육지 출하 최적화

### 8.2 2차 산업 — 제조업 부품 공급망

- 생산 계획 연동으로 JIT(Just-In-Time) 자동 보충
- 단납기 주문 발생 시 대체 공급사 0.8ms 내 탐색
- 반도체·전기차 부품 등 고가 화물 OpenHash 불변 추적

### 8.3 3차 산업 — 유통·소비재

- 주문 폭증 시 마이크로 창고 실시간 활성화
- 평균 배송 시간 **4시간 단축**
- 이커머스 라스트마일 최적화

### 8.4 공공 — 의료·방역·국가 비축

- K-911·K-Health 연동으로 긴급 의약품 30분 내 병원 도착
- 국가 비축물자 실시간 재고 현황 제공
- 혈액·백신 등 극저온 물류 IoT 온도 모니터링

## 10. 보안 및 개인정보

- **PDV 오프라인 저장**: 개인 데이터는 사용자 기기에 저장, 대규모 해킹 불가
- **OpenHash 불변 기록**: 배송 증빙 위·변조 불가
- **차분 프라이버시 (ε ≤ 0.1)**: 통계 분석 시 개인 식별 불가
- **운전자 신원 마스킹**: 성+○○ 형식 (예: 김○○)
- **PDV L1 인증 필수**: 실명 인증 차량만 매칭 대상
